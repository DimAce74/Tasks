1) Название класса написано с маленькой буквы.
2) Поле listElements должно быть не финальным (есть два метода, изменяющих это поле) и может быть приватным (для поля есть public геттер/сеттер).
3) Использование HashSet для хранения массивов нецелесообразно, так как массивы имеют унаследованный от Object метод equals, который проводит сравнение только по ссылке. Значит Set сможет содержать одинаковые по значению элементы.
4) Если listElements все-таки Set, то проверка на уникальность при добавлении не нужна.
5) Метод LogData - название метода должно начинаться с маленькой буквы.
6) Метод LogData - так как предполагается его использование из внешних модулей, метод должет быть public.
7) Метод LogData - в методе идет конкатенация строк, значит эффективнее использовать StringBuilder.
8) Метод LogData - в методе не используется передаваемый в качестве аргумента separator.
9) Метод compareNumberWithZero - учитывает не все возможные варианты вызовов.
10) Метод getDatePlus5WorkDaysPlus1DayPlus1Year - методы getYear(), getMonth(), getDay() - Deprecated.
